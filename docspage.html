Welcome to the EUP developer documentation. This guide covers everything you need to integrate EUP into your product, automate prompt workflows, and build AI-powered experiences with precision.

Quick Start API Reference SDKs Changelog

WHAT IS EUP? §

EUP (Excessive User Prompting) is a prompt engineering platform that eliminates trial-and-error when working with large language models. Instead of iterating endlessly on prompt phrasing, EUP gives you a structured, visual toolkit to build, test, version, and deploy prompts at scale.

⚡
Prompt Builder
Visual drag-and-drop editor with variables, conditionals, and live preview.
📚
Prompt Library
Thousands of community templates. Fork, version, publish.
🔗
REST API
Full API access to run, manage, and analyse prompts programmatically.
🤝
Teams
Role-based access, shared workspaces, version history.

QUICK START §

Get up and running in under 5 minutes. You'll need an EUP account and an API key to follow along.

  1. Create an account

    Sign up at eup.io/signup — the free tier includes 50 prompts per month with no credit card required.

  2. Generate an API key

    Navigate to Settings → API Keys in your dashboard and click Generate New Key. Store it securely — it's shown only once.

  3. Install the SDK

    Choose your preferred language below and install the official EUP client library.

  4. Run your first prompt

    Use the example below to execute a prompt from your library via the API in seconds.

terminal
# Install via npm
npm install @eup-io/client
# Install via pip
pip install eup-python
# Install via yarn
yarn add @eup-io/client

Your First API Call

javascript
import { EUPClient } from '@eup-io/client';

const eup = new EUPClient({
  apiKey: 'eup_live_xxxxxxxxxxxxxxxxxxxx',
});

const result = await eup.prompts.run({
  promptId: 'prompt_abc123',
  variables: {
    topic: 'prompt engineering',
    tone:  'professional',
    length: 300,
  },
  model: 'gpt-4o',
});

console.log(result.output);
// → "Prompt engineering is the practice of..."
from eup import EUPClient

eup = EUPClient(api_key="eup_live_xxxxxxxxxxxxxxxxxxxx")

result = eup.prompts.run(
  prompt_id="prompt_abc123",
  variables={
    "topic": "prompt engineering",
    "tone":  "professional",
    "length": 300,
  },
  model="gpt-4o",
)

print(result.output)
# → "Prompt engineering is the practice of..."
curl -X POST https://api.eup.io/v2/prompts/run \
  -H "Authorization: Bearer eup_live_xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "promptId": "prompt_abc123",
    "variables": {
      "topic": "prompt engineering",
      "tone": "professional",
      "length": 300
    },
    "model": "gpt-4o"
  }'
💡
Pro tip

Use environment variables to store your API key. Never commit eup_live_* keys to version control. Use eup_test_* keys in development — they don't count against your quota.

PROMPT BUILDER §

The Prompt Builder gives you a visual workflow for composing reusable prompts with variables, constraints, and output schemas. Every save creates a version so you can diff and roll back safely.

Builder Workflow

  1. Define your intent and context

    Use a clear system block that states objective, audience, and non-negotiable constraints.

  2. Insert variables

    Reference variables with {{variable_name}} syntax and provide validation rules in config.

  3. Preview with sample inputs

    Run quick tests against multiple models before publishing to production environments.

PROMPT LIBRARY §

The library stores team and personal templates with tags, ownership, and usage analytics. Fork community templates and adapt them to your own workflows in one click.

CapabilityDescription
Version HistoryTrack changes per prompt with timestamps and editor attribution.
CollectionsGroup prompts by product area, team, or customer segment.

TESTING & PREVIEW §

Run side-by-side evaluations to compare output quality, latency, and token cost. Evaluation runs can be exported for QA review and release approvals.

ℹ️
Regression testing

Store golden inputs and expected patterns, then re-run after any prompt or model change to catch output drift early.

AUTHENTICATION §

EUP uses Bearer token authentication for all API requests. Include your API key in the Authorization header of every request.

⚠️
Keep your keys secret

API keys carry full account privileges. Rotate compromised keys immediately from the dashboard under Settings → API Keys → Revoke.

Key Types

Key PrefixTypeDescription
eup_live_stringProduction key. Counts against quota. Use in deployed apps.
eup_test_stringTest key. Returns mock responses. Free for CI/CD pipelines.
eup_team_stringTeam-scoped key. Bound to workspace permissions only.

API ENDPOINTS §

All endpoints are prefixed with https://api.eup.io/v2. Responses are always JSON. Errors follow the standard error format.

POST /prompts/{promptId}/run Execute a prompt
Execute a saved prompt with variable substitution. Returns the model's generated output along with token usage and latency metadata.
GET /prompts List all prompts
Returns a paginated list of all prompts in the authenticated workspace, sorted by last modified descending.
POST /prompts Create a prompt
Creates a new prompt in the workspace. Accepts a name, body template with {{variable}} syntax, and optional metadata tags.
PUT /prompts/{promptId} Update a prompt
Updates an existing prompt. Creates a new version in the history automatically. Previous versions remain accessible via the versions endpoint.
DELETE /prompts/{promptId} Archive a prompt
Archives (soft-deletes) a prompt. Archived prompts are retained for 30 days and can be restored from the dashboard.

Request Parameters — /prompts/:id/run

ParameterTypeRequiredDescription
promptId string required The unique identifier of the prompt to run. Found on the prompt detail page.
variables object optional Key-value map of variable names to values. Must match variables declared in the prompt template.
model string optional Override the default model. Accepted: gpt-4o, claude-3-5-sonnet, gemini-2.0-flash.
stream boolean optional When true, returns a streaming SSE response. Defaults to false.
maxTokens integer optional Maximum output tokens. Overrides the prompt-level setting. Range: 1 – 32768.

json

RATE LIMITS §

EUP enforces per-API-key rate limits to ensure stability for all users. Limits are applied on a sliding 60-second window.

PlanRequests / minTokens / dayConcurrent
Free60100K5
Pro6005M50
EnterpriseCustomUnlimitedUnlimited
ℹ️
Rate limit headers

Every response includes X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset headers so you can implement backoff strategies.

ERROR CODES §

EUP uses standard HTTP status codes. All error responses have a consistent JSON body with a machine-readable code field.

json — error response
{
  "error": {
    "code":     "PROMPT_NOT_FOUND",
    "message":  "Prompt 'prompt_abc123' does not exist in this workspace.",
    "status":   404,
    "requestId": "req_7hYm2LpXwN3q"
  }
}
🚨
Breaking change in v2

The error.code field replaces the legacy errorCode field from v1. Update your error handling if migrating from v1.

WEBHOOKS §

Subscribe to delivery and run lifecycle events to keep your systems in sync. Webhooks are signed with HMAC-SHA256 using your workspace secret.

POST /webhooks/events Run event payload
Event types include run.completed, run.failed, and prompt.published. Retry policy uses exponential backoff for up to 24 hours.
